## Українська версія

Обчисліть календарну дату за номером дня в році.
Користувач задає день року у форматі YYYY, DDD, наприклад: 2024, 256. Програма має вивести дату у форматі YYYY-MM-DD.

Знайдіть усі наявні code smells і прокоментуйте їх назви у відповідних рядках коду.
Виконайте рефакторинг і прокоментуйте застосовані методи.
Переконайтеся, що рішення працює коректно, а юніт-тести проходять успішно.
Перепишіть код відповідно до ООП-парадигми. Прокоментуйте всі внесені зміни.
Зробіть стільки кроків, скільки потрібно, щоб відобразити процес рефакторингу.

## English version

Calculate the calendar date from the day number within a year.
The user provides a day of the year in the format YYYY, DDD, for example: 2024, 256. The program should output the date in the format YYYY-MM-DD.

Identify all existing code smells and annotate their names on the relevant lines of code.
Refactor the code and comment on the techniques you applied.
Ensure the solution works correctly and that all unit tests pass.
Rewrite the code according to OOP principles. Comment on every change you make.
Take as many steps as necessary to clearly demonstrate the refactoring process.


## Wersja polska

Oblicz datę kalendarzową na podstawie numeru dnia w roku.
Użytkownik podaje dzień roku w formacie YYYY, DDD, na przykład: 2024, 256. Program powinien zwrócić datę w formacie YYYY-MM-DD.

Zidentyfikuj wszystkie istniejące code smells i oznacz ich nazwy przy odpowiednich liniach kodu.
Przeprowadź refaktoryzację i skomentuj zastosowane techniki.
Upewnij się, że rozwiązanie działa poprawnie oraz że wszystkie testy jednostkowe przechodzą pomyślnie.
Przepisz kod zgodnie z zasadami OOP. Skomentuj każdą wprowadzoną zmianę.
Wykonaj tyle kroków, ile potrzeba, aby wyraźnie pokazać proces refaktoryzacji.
